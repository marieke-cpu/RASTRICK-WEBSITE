§ 01 / The Name

Where it comes from.

RASTRICK
is my
maiden
last name.

I kept it. Not just for sentiment — though that's part of it — but because a name that makes people pause, even briefly, tends to be remembered. It's sharp. A little unexpected. Impossible to forget. Everything a business name should be.

There's something intentional about building a studio under a name that's genuinely yours. Not a made-up word, not a studio name chosen from a list — my actual name, the one I was given and chose to carry forward. Every project carries it. Every site that launches into the world carries it. That's not a small thing to me.

It also set the tone for everything else. If the brand is going to be authentic, it starts at the name. If the work is going to feel personal, so does the studio. RASTRICK. — the full stop is intentional. It's a statement, not an afterthought.

§ 02 / The Story

How I got here.

Origin story

Design
found me
first.

It started long before websites. It started with noticing things — the way a logo communicates something before you've consciously read it. The way a font carries weight, personality, mood. The gap between something that looks like it belongs and something that doesn't, and why that gap matters more than most people realise.

"I wanted to design things that didn't just look good — they had to work."

Code came when I realised that designing something and handing it over meant losing the details that mattered. The micro-interaction that made something feel right. The spacing that was 2px off. The animation timing that changed the whole mood of a page. I needed to build what I designed.

So I learned. HTML first — the skeleton of everything. CSS to make it breathe. JavaScript to make it move and respond. Not from a course that handed me a framework and told me I was a developer — from actually building things, breaking them, understanding what was happening at the code level, and building them better.

There's no shortcut to understanding how the web actually works. Page builders abstract it away. Templates hide it. Writing code from scratch forces you to know it — and knowing it means you can do things that builders simply can't. Custom interactions. Pixel-precise layouts. Performance that actually loads fast because there's nothing there that doesn't need to be.

That's the studio now. Design that's built to work the way it was designed. No handoff gap. No details lost in translation. The person who drew the layout is the same person writing the code.

§ 03 / Philosophy

No templates.
ever.

Most websites look like most other websites. Not because the businesses are the same — they're not — but because the tools used to build them are. The same twelve layouts. The same drag-and-drop structure. The same bloated code underneath. Swap out a logo, change a colour, call it a "custom site." It isn't.

Every RASTRICK website is written from a blank file. HTML, CSS, and JavaScript — no page builders, no WordPress, no shortcuts. Code I write, I control. Details I design, I build. What you get isn't a template with your name on it. It's a website built specifically for your business, your goals, and your audience.

Small businesses deserve websites that work as hard as they do. Not an off-the-shelf solution that was designed to fit everyone and fits no one particularly well. Something sharp, fast, and built to last — and built to be unmistakably yours.

That means the conversations matter too. No account managers sitting between us. You speak directly to the person making decisions about your site. Every call, every brief, every revision — that's me, and it stays that way for the life of the project.
